## 大屏一键注销数字钥匙故障分析

### 流程概述

```
大屏 → TBOX → One Backend → 数字钥匙管理平台 → JVSP / 手机厂商后台
```

**关键步骤**：
1. 大屏发起注销请求，生成 4 位验证码
2. TBOX 发送短信验证码至主账户手机号
3. 用户输入验证码校验
4. 校验通过后，TBOX 删除对应 VIN 的所有数字钥匙
5. 同步注销状态至 JVSP 和手机厂商后台

### 常见故障场景

#### 场景1：弹窗显示手机号与实际不符

**现象**：弹窗显示的手机号不是主账户真实手机号

**可能原因**：
- 数字钥匙应用获取手机号来源错误（获取了测试账号 / 默认账号）
- 账号服务缓存了错误的账号信息
- 主账户手机号未正确同步到数字钥匙服务

**排查要点**：
1. 调用账号服务 API 获取主账户真实手机号，对比弹窗显示
2. 检查数字钥匙应用获取手机号的接口调用链路
3. 检查是否存在测试账号残留配置
4. 核对大屏账号与数字钥匙账号是否一致

#### 场景2：验证码发送失败

**现象**：主账户未收到验证码短信

**可能原因**：
- 手机号获取失败导致短信发送目标错误
- TBOX 短信服务异常
- 网络问题导致短信发送超时

**排查要点**：
1. 检查 TBOX 日志中的短信发送请求
2. 检查目标手机号是否正确
3. 检查短信网关响应

#### 场景3：验证码校验失败

**现象**：输入正确验证码后提示失败

**可能原因**：
- 验证码不匹配（超时或生成错误）
- 密钥验证逻辑异常
- TBOX 与后台同步问题

### 关键日志搜索词

**大屏侧**：
- `account` — 账号服务
- `注销` / `删除钥匙` / `delete key` — 注销流程
- `手机号` / `phone` / `mobile` — 手机号相关
- `API` / `HTTP` — 接口调用

**TBOX / 车端**：
- `删除钥匙` / `delete key` — 删除操作
- `验证码` / `SMS` — 短信相关
- `VIN` — 车辆标识

**后台服务**：
- `One Backend` / `数字钥匙管理平台` — 后端服务
- `JVSP` — 账号服务
- `KeyID` — 钥匙 ID

### 故障定位检查清单

| 检查项 | 检查内容 | 归属端 |
|--------|----------|--------|
| 主账户手机号 | API 返回的是否为真实手机号 | 账号服务 |
| 手机号来源 | 数字钥匙应用从哪个接口获取手机号 | 大屏应用 |
| 验证码发送 | TBOX 是否成功发送短信 | 车端 TBOX |
| 钥匙删除 | 删除接口是否成功调用 | 后台服务 |
| 状态同步 | JVSP / 手机厂商后台是否同步 | 后台服务 |

## CAN 日志分析方法

### 数字钥匙相关 DBC 文件优先级

| 优先级 | DBC 文件 | 说明 |
|--------|---------|------|
| ⭐⭐⭐ | **ICAN_CEA_EP07_20250606.dbc** | 内部 CAN，数字钥匙核心数据 |
| ⭐⭐⭐ | **ECAN_CEA_EP07_20250606.dbc** | 动力 CAN |
| ⭐⭐⭐ | **LPCAN_CEA_EP07_20250606.dbc** | 低压 CAN |
| ⭐⭐ | **BCAN_CEA_EP07_20250606.dbc** | 车身 CAN，网络管理 / Ready 状态 |

### 时间戳定位

1. ASC 日志开头包含日期时间：`date Fri Jan 30 02:03:44 pm 2026`
2. 计算相对时间：测试时间 - 日志开头时间
3. 在 ASC 中搜索对应时间戳附近的 CAN 消息

### 时区转换

- 日志时间通常为本地时间
- 北京时间 (CST) = UTC + 8

## TBOX 环境配置（调试重点）

### 环境 URL 配置

| 环境变量 | 含义 | 调试命令 |
|----------|------|----------|
| `TBOX_ACCOUT_REQ_URL_PATH` | 车企后台接口环境选择 | `0x00` = Prod，`0x01` = UAT |

### 查看/设置

```bash
# 查看当前环境
test_remote_send NVRAM | grep TBOX_ACCOUT_REQ_URL_PATH

# 设置为 UAT 环境
test_remote_send NVRAM_SET TBOX_ACCOUT_REQ_URL_PATH 0x01
```

### ⚠️ 常见问题

- TBOX 环境配置错误 → 数字钥匙请求打到错误的服务器
- Prod/UT 环境混淆 → 钥匙无法与车辆正确关联
- 联调时应先确认 TBOX 环境与测试环境一致
